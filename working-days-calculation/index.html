Pipeline 1 — Fault Browser Files
How it works:
  • Select one or more Fault Browser Excel exports (the 5-row header is auto-skipped)
  • Files are merged and sorted by Started at timestamp
  • ATM state is looked up from DB Master using Terminal ID
  • Downtime is split into: Post Banking Hours / Weekend / Public Holiday / Actual
📂 Click to select one or more .xlsx files
Expected columns: Terminal ID, Fault, Started at, Ended at, Duration (minutes)

Pipeline 2 — Own Fault File
How it works:
  • Upload your own fault data file (e.g. Book15.xlsx → Sheet1 is used)
  • State is taken from the file directly (or looked up from DB Master if missing)
  • No merging — calculated columns are appended to the original data
